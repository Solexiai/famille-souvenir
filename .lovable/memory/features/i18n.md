## Architecture

- `src/i18n/types.ts` — Translations interface, TerminologyPack, SupportedLanguage, CountryGroup, JurisdictionPack types
- `src/i18n/locales/fr.ts`, `en.ts`, `es.ts` — Full translation files
- `src/i18n/terminology.ts` — Terminology packs: quebec, canada_general, us_general, latam_general
- `src/i18n/jurisdictions.ts` — Region lists (provinces, states, LATAM countries), resolveJurisdiction()
- `src/contexts/LocaleContext.tsx` — LocaleProvider + useLocale() hook

## Key rules
- Jurisdiction stored on family_circles (dossier-level), user preference on profiles
- Language and jurisdiction are independent — user can pick any combo
- latam_general = explicitly general, never implies unified legal framework
- Terminology pack overrides legal terms (executor vs liquidateur vs albacea)
- localStorage keys: solexi_lang, solexi_jurisdiction, solexi_country_group
